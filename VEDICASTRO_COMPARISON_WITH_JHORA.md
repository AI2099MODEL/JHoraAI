## 2. Key Mathematical Audits

### A. Ayanamsa (Sidereal Shift)
*   **Ayanamsa Accuracy:** Both systems query the underlying Swiss Ephemeris library to calculate the ayanamsa value (sidereal offset) for the target birth/event moment.
*   **Result:** Longitudes in both systems are perfectly synchronized to the second of an arc ($1/3600^{\circ}$), assuming the same ayanamsa (e.g. Krishnamurti) is chosen.

### B. House Cusps (Placidus)
*   **Calculation:** For KP Astrology, the Placidus house system is mandatory. Both JHora and VedicAstro calculate Placidus boundaries using the local birth coordinates and birth time, adjusting for sidereal time.
*   **Result:** Cusp boundary degrees align perfectly, allowing the same planets to fall in identical houses in both systems.

### C. Star Lord & Sublord Divisions (KP 1-249)
*   **VedicAstro's Division Algorithm:**
    ```python
    deg = deg - 120 * int(deg / 120)
    # Recursively subdivides each 13°20' Nakshatra into 9 parts proportional to Vimshottari Dasha years
    ```
    This matches the exact logic used in JHora to compute sublords.
*   **VedicAstro's Sub-Sublord Division:** Performs the third-level recursion (e.g. dividing sublord spans into 9 sub-sublord spans based on Vimshottari proportions), matching JHora's detailed planetary grids.

## 3. Discovered Variations (and Resolution)

While the astronomical coordinates are identical (due to sharing the Swiss Ephemeris core), minor differences can arise from input configuration parameters:

1.  **Geocentric vs. Topocentric Positions:**
    *   *JHora:* Defaults to Geocentric positions for planets.
    *   *VedicAstro:* Uses Geocentric positions by default.
2.  **UTC Offset Formatting:**
    *   *JHora:* Expects local timezone offsets.
    *   *VedicAstro:* Resolves timezones dynamically using `timezonefinder` based on latitude and longitude, but accepts manual `utc` offsets. Ensure the correct UTC offset (e.g., `+05:30` for India, `-04:00` for Eastern Daylight Time) is sent to the API.
3.  **Rahu/Ketu Calculation Mode:**
    *   *JHora:* Allows selection between "Mean Node" and "True Node".
    *   *VedicAstro:* Flatlib default is "True Node". Ensure JHora is set to "True Node" for exact coordinate matches.
